# デザインシステム詳細（色覚配慮パレット・色↔記号）

> CLAUDE.md「ステータス色・記号」から退避した参照詳細。
> 値の**正本はコード**（`shared.css` の `:root` と `tags.js` の `STATUS_TAG_MARK`）。
> ここはその背景・根拠のメモ。実装時はコード側を真とする。

## ステータス色パレット（D/P型 赤緑色盲対応）

ステータス色（黄/緑/灰/青）は **D/P型でも判別可能**なパレットを使う。
ペール（背景）+ 濃い枠 + 濃い文字の3層構成で、明度差と hue family の両方を保つ：

| 色 | 背景 (bg) | 枠 | 文字 |
|---|---|---|---|
| 黄 (yellow) | `#fef3c7` | `#d97706` (amber-600) | `#92400e` |
| 緑 (green) | `#dcfce7` | `#16a34a` (green-600) | `#166534` |
| 灰 (gray) | `#9ca3af` | `#374151` | `#111827` |
| 青 (blue) | `#bfdbfe` | `#2563eb` | `#1e3a8a` |

- 緑は旧 teal-500 系から純緑系へ（ペール化で teal だと青と混ざるため）。
- 灰は「終了済みで目立たせたくない」用途のため bg を gray-200 → gray-400 に一段濃く（3層構造は維持）。
- ステータスボタン内には**形マーク**を入れ、色だけに依存しない。一覧UIなどで形マークが邪魔な場合は色のみ可（明度差で識別可能なため）。

## 色↔記号の正準マッピング（必ず1:1で統一）

形マークは全UI（タグ選択ピッカー、患者ステータス選択ポップアップ、ステータスボタン、一覧チップ）で**同じ記号**を使う。実装は `tags.js` の `STATUS_TAG_MARK` を単一ソースとし、ここからずらさない：

| 色 | 記号 | 意味 |
|---|:--:|---|
| 白 (none) | − | なし / 未 |
| 黄 (yellow) | ▲ | 要注意 / 未 |
| 緑 (green) | ✓ | 済 |
| 灰 (gray) | ✕ | 終了 / 対象外 |
| 青 (blue) | ★ | 新着 / 注意 |

- **白 (none) も記号 `−` を持つ**（v8.7）。白アイコンは無印だと「空欄か未設定か」が分かりにくく、特に白背景のポップアップ上で視認性が悪いため明示的に `−`(なし)を描く。
- **青に `+`（十字）は使わない**（v8.4）。「十字（+形）アイコンは使わない」ルールに抵触し視認性も悪いため `★` を採用。
- 記号を淡背景に重ねる時は**濃い文字色（各色の「文字」値）**で描き、白抜きで潰さない（特に青の淡背景＋薄記号は不可）。選択ポップアップのスウォッチは濃色背景＋白記号でも可（コントラストが取れる方）。
