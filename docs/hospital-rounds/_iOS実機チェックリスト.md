# iOS 実機チェックリスト（開発者・配布担当者向け）

> このリストは **自動テストでは確認できない iOS/Safari 固有の挙動** を、実機（配布予定の iPad モデル）で人が確認するためのものです。
> 自動E2E（`npm run test:e2e`）はロジック・画面遷移・データ整合を Chrome で検査しますが、以下は **実機の Safari でしか分かりません**。
> MDM 配布の本番判断の前に、少なくとも1台で通してください。

前提: **Safari で開く → 共有 → ホーム画面に追加** で PWA としてインストールした状態で確認する（Chrome ではない）。

---

## A. データの永続性（最重要）⚠️
全患者データは端末内の IndexedDB に保存されます。iOS はストレージを退避（eviction）することがあるため、**消えないか**を必ず確認します。

- [ ] サンプルデータ（`test/sample-data/comprehensive.device-archive.json`）を取り込む
- [ ] **アプリを完全終了 → 再起動** してもデータが残る
- [ ] **端末を再起動** してもデータが残る
- [ ] **数日（最低3日、できれば1〜2週間）放置**後に開いてもデータが残る ← iOS eviction の本命確認
- [ ] ストレージ残量が少ない状態でも消えないか（可能なら端末を圧迫して確認）
- [ ] 設定 →（必要なら）「永続化」要求が効いているか（`navigator.storage.persist()` は best-effort）
- [ ] 結論として **JSON バックアップ運用（[09]）を必須手順として周知**する

## B. PWA インストールとカメラ（QRスキャン）
- [ ] Safari からホーム画面に追加でインストールできる（本番アイコン／テスト版アイコンが正しい）
- [ ] **ホーム画面アイコンから起動**（standalone）した状態で、QR スキャンの **カメラが起動する**
  - iOS のインストール済み PWA はカメラ制限の歴史があるため、ブラウザ内ではなく **アイコン起動**で要確認
- [ ] カメラ権限の許可ダイアログが出て、許可後に読み取れる
- [ ] 実際の QR（ホーム／プロブレムリスト／共有／設定／フォーマット）を**別端末と往復**して読み取れる

## C. ブラウザ機能差
- [ ] **バイブ（`navigator.vibrate`）は iOS 非対応** → 震えないが**クラッシュしない**ことだけ確認（無害）
- [ ] QR の暗号化＋圧縮（CompressionStream）: 配布する iOS バージョンが Safari 16.4+ か。古い場合はフォールバック（E1）で読めるかを別端末往復で確認
- [ ] 電子カルテ転記用 QR（平文）が標準カメラ（Windows 11 など）／実機で読める

## D. バックアップ / 復元（iOS のファイル挙動）
- [ ] 「JSON 書き出し」→ **「"ファイル"に保存」**で保存できる
- [ ] 保存した JSON を「JSON 取り込み」「端末まるごと取込」で復元できる
- [ ] ファイル名・日付が正しい

## E. 表示・操作（iPad Safari の見た目）
- [ ] ヘッダー「ユーザー名 / 病棟 ▾」が崩れない、セーフエリア（ノッチ・ホームバー）に被らない
- [ ] 入力欄の自動拡大（field-sizing 未対応時のフォールバック）が効く
- [ ] タッチ操作ターゲットが 44×44 を満たし、押しやすい
- [ ] 縦/横回転で崩れない
- [ ] 印刷（電子カルテ転記の補助）が必要なら、その出力

## F. ユーザー機能（複数人運用）
- [ ] ユーザー切替でデータがまるごと入れ替わる
- [ ] 同名ユーザーが拒否される
- [ ] ユーザー削除でその人の全病棟・設定が消える（誤操作防止の確認文言）
