# iOS / MDM 導入ガイド（病院IT担当向け）

回診アプリ（hospital-rounds）を院内端末へ配布・運用するための手引き。
**医療カテゴリ固有**の内容（MDM・臨床データ運用）。動作環境の共通ベースライン
（オフライン / ホーム画面追加 / iOS永続性 / バックアップ）は snishi-code.com の
「動作環境」節が正本。ここはそれを病院運用の文脈で具体化したもの。

## 前提：このアプリのデータの持ち方

- 患者データはすべて**端末内（IndexedDB）にのみ**保存され、外部送信は一切ない（機械ガードで担保）。
- 利点：情報が外に出ない。**裏返しの注意点**：データは端末に閉じるので、端末側の永続性とバックアップが運用の要。

## iOS（Safari）で必ず守ること

1. **「ホーム画面に追加」して、アプリとして起動する（必須）**
   - Safari の**タブのまま**使うと、iOS のストレージ退避により**約7日**の未使用でローカルデータが消去対象になる。
   - **ホーム画面に追加した Web アプリはこの7日消去の対象外**になり、永続性が大きく改善する。
   - したがって「タブ運用は不可、全端末でホーム画面追加」を運用ルールにする。
2. **長期間の未使用・ストレージ逼迫に注意**
   - ホーム画面追加後も、端末のストレージが極端に逼迫すると OS がデータを解放する可能性はゼロではない。
   - → 後述のバックアップで保険をかける。
3. **検証（導入前に必ず）**
   - テスト端末（iPad）で「ホーム画面追加 → データ入力 → 数日放置 → データ生存」を確認してから本運用へ。

## MDM での配布

院内端末は MDM 管理を前提とする。Safari の **Web Clip（ホーム画面ショートカット）** を MDM から配布すると、
「全端末で確実にホーム画面追加された状態」を中央管理でき、上記1を運用で徹底できる。

- 配布する URL は本番 `https://medical.snishi-code.com/hospital-rounds/`。
- Web Clip は「フルスクリーン（standalone）」で開く設定にする（アプリとして起動）。
- 具体的な手順は採用する MDM 製品（Intune / Jamf / 各社 MDM）に依存するため、製品側のドキュメントに従う。
  - ※ MDM 自体は WebKit のストレージ退避仕様を変えるものではない。MDM の役割は「確実な home-screen 配布＋端末管理」。
- より強い永続性・管理保証が必要になった場合の次の手は、ネイティブラッパー（MDM 管理アプリ化）。現時点では Web Clip + 運用で足りる想定。

## バックアップ運用（臨床データの最後の砦）

端末の故障・初期化・機種変更・OS のデータ解放に備え、**定期バックアップを運用ルール化**する。

- アプリの**エクスポート機能（QR / JSON）**で書き出す。
- 頻度・保管場所（誰が・どこへ）は各院の運用に合わせて定める。
- 復元手順もあらかじめ確認しておく。

## Android（参考）

- 開発者の主環境は Android（Chrome）。Chrome は Safari ほど積極的なローカルデータ退避をしないため、永続性は比較的堅い。
- 院内導入で iOS を選ぶのは「IT担当の慣れ・MDM運用のしやすさ」という運用都合であり、永続性の技術的優位ではない点に留意（だから iOS では上記1のホーム画面追加が必須）。
